## 快速开始

### 首次使用

```bash
# 1. 启动您的智能体
opc200 start

# 2. 初始化 Journal（只需一次）
/journal init

# 3. 开始对话
你好，我是 OPC-001，请帮我规划今天的任务
```

### 基本命令

| 命令 | 功能 | 示例 |
|------|------|------|
| `/journal` | 查看/管理 Journal | `/journal today`, `/journal week` |
| `/task` | 创建异步任务 | `/task 分析竞品报告，明天早上给我结果` |
| `/pattern` | 查看行为模式 | `/pattern work-style` |
| `/milestone` | 里程碑追踪 | `/milestone list` |
| `/insight` | 获取洞察 | `/insight weekly` |
| `/agent` | 切换 Agent | `/agent research`, `/agent dev` |
| `/help` | 获取帮助 | `/help journal` |

### 长期记忆

```

OPC200: 基于 30 天的 Journal 记录：

工作风格画像：
• 高效时段: 上午 9-11 点（产出最高）
• 偏好模式: 深度工作 2 小时 + 休息 15 分钟

## 多 Agent 协作

### Agent 类型

| Agent | 专长 | 调用方式 |
|-------|------|---------|
| 主 Agent | 日常协调、综合任务 | 默认 |
| Research | 信息搜集、竞品分析 | `/agent research` |
| Dev | 代码开发、技术实现 | `/agent dev` |
| Content | 内容创作、文案 | `/agent content` |
| Data | 数据分析、报告 | `/agent data` |

### 协作示例

```
您: 帮我准备投资人路演材料

📋 任务分解：
├─ Research Agent: 搜集投资人背景、关注点

├─ Content Agent: 撰写演讲稿、故事线
└─ 主 Agent: 整合所有材料，统一风格

预计 1 小时完成。您可以：
• 实时查看各 Agent 进度
• 对中间结果提出修改
• 指定特定 Agent 重新执行

开始执行？(是/调整/取消)
```

## 数据与隐私

### 数据控制

**本地部署客户**：
- 所有数据存储在您本地
- 支持中心仅在有授权时可访问
- 可随时导出全部数据

**云端托管客户**：
- 数据加密存储（AES-256）
- 定期自动备份
- 可随时请求数据导出或删除

### 隐私设置

```
您: /privacy

OPC200: 隐私设置

当前配置：
• 记忆保留: 永久（可随时删除）
• 数据同步: 启用
• 分析参与: 已退出
• 第三方共享: 已禁用

可选操作：
• 导出所有数据
• 删除特定记忆
• 清除全部历史
• 调整隐私级别
```

## 故障排除

### 常见问题

**Agent 响应慢**：
```
您: /status

OPC200: 系统状态：
• Gateway: 正常
• 网络: 延迟 150ms（稍高）
• 当前负载: 中等

建议：当前有 3 个异步任务运行中，
响应可能稍慢。重要任务？我可以优先处理。
```

**记忆不准确**：
```
您: 那不是正确的

OPC200: 抱歉，让我纠正。

原记录: [错误信息]
更正为: [正确信息]

已更新记忆，未来会引用正确版本。
还有其他需要纠正的吗？
```

```

...

• opc-security-guard: 等待更新

建议运行: /skill update opc-security-guard

## 快捷键与技巧

### 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Enter` | 发送消息 |
| `Ctrl+/` | 显示命令列表 |
| `Ctrl+↑` | 浏览历史 |
| `Ctrl+Shift+J` | 快速打开 Journal |

### 高级技巧

1. **引用上下文**: 使用 `>` 引用之前的对话
2. **多行输入**: 使用 ` ``` ` 包裹代码块
3. **快速任务**: `@research 查询...` 直接指定 Agent
4. **紧急标记**: 消息开头加 `!` 表示紧急
